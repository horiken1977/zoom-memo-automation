1. システム概要

Zoom Memo Automationは、Zoomミーティングの内容を自動的に記録・整理し、議事録を生成するシステムです。

主要目標

  • Zoomミーティングの自動録音・録画
  • 音声からテキストへの変換（STT: Speech-to-Text）
  • AIによる議事録の自動生成
  • 参加者への自動配信

2. 機能一覧

F001: Zoom連携機能

説明: ZoomのWebhookを利用してミーティングの開始・終了を検知し、自動的に録音・録画を開始する機能

優先度: 高

状態: 計画中

詳細仕様

項目 内容
入力 Zoom Webhook イベント
出力 録音・録画データ
処理 ミーティング検知、録音開始、メタデータ保存

F002: 音声認識機能

説明: 録音された音声データを高精度でテキストに変換する機能

優先度: 高

状態: 計画中

詳細仕様

項目 内容
入力 音声ファイル（MP3/WAV/M4A）
出力 テキストデータ（JSON形式）
API OpenAI Whisper API / Google Speech-to-Text

F003: 議事録生成機能

説明: 変換されたテキストをAIで解析し、構造化された議事録を自動生成する機能

優先度: 高

状態: 計画中

詳細仕様

項目 内容
入力 会議テキストデータ、参加者情報
出力 構造化議事録（Markdown/HTML）
処理 要約生成、アクションアイテム抽出、発言者識別

F004: 配信機能

説明: 生成された議事録を参加者に自動配信する機能

優先度: 中

状態: 計画中

詳細仕様

項目 内容
配信方法 Email、Slack、Teams
配信タイミング ミーティング終了後30分以内
形式 PDF、HTML、Markdown

3. 非機能要件

性能要件

  • 音声認識処理時間: 録音時間の1/4以下
  • 議事録生成時間: 5分以内
  • 同時処理可能ミーティング数: 10セッション

セキュリティ要件

  • 録音データの暗号化保存
  • アクセス権限の管理
  • GDPR準拠のデータ処理

可用性要件

  • 稼働率: 99.5%以上
  • 障害復旧時間: 30分以内

4. 技術要件

開発技術スタック

  • フロントエンド: Next.js, React, TypeScript
  • バックエンド: Node.js, Express
  • データベース: PostgreSQL, Redis
  • AI/ML: OpenAI API, Whisper
  • インフラ: Vercel, AWS S3
